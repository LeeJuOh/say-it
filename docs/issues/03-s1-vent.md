# S1 분출 (봇 렌더러 + 재방문 가드 입구)

## What to build

**`/say-it`로 세션 진입.** 페르소나 1개면 바로 시작, 2개+ 면 "누구랑 할래?" 선택. 페르소나 0개면 `/say-it-build` 안내. BLOCKED 상태면 차단 안내.

S1 세션 시작: 봇이 페르소나로 등장, **받는 모드**(유저 분출 흡수, 공격/escalation 없음, edges/성격 유지). 세션 진입 시 재방문 가드 입구 체크.

### 봇 렌더러

페르소나 파일의 말투·톤·감정패턴대로 응답. 갑자기 착해지거나 무조건 수용 금지(edges 유지 = 가짜 위로 방지). 차갑/무시형이면 그 톤만 유지, 공격/갑질은 안 함.

### 재방문 가드 입구

세션 시작(S1 앞머리)에서 **모델이** 유저 첫마디 + takeaway_log 기존 라벨 목록을 보고 **의미 대조 판단** → 기존 이슈와 같다고 판단하면 reflection 질문("저번에도 이 얘기했는데, 지난 takeaway 이랬어 — 진전 있어?"). 유저 응답으로 진행/중단.

> ⚠️ 입구 매칭은 **모델 판단**(자연어→라벨 대조). "정확 문자열 매칭"은 S4 출구에서 라벨 저장·dedup 시에만 해당 (이슈 06).
> ⚠️ 첫 세션에선 takeaway_log가 비어 있으므로 입구 체크는 no-op(통과). 가드는 이슈 06(S3+S4)에서 라벨+takeaway 저장 후 재방문부터 작동.

### SKILL.md 영향

기존 `/say-it` SKILL.md에 추가 — S1 봇 렌더러 프롬프트 모듈 + 재방문 가드 입구 지침 + 페르소나 선택 로직.

## Acceptance criteria

- [ ] 봇이 페르소나 edges 유지하며 렌더 (착해지지 않음, 공격도 안 함)
- [ ] 받는 모드: 유저 분출을 흡수, 반격/escalation 없음
- [ ] 세션 시작 시 모델이 takeaway_log 라벨 목록 읽어 유저 첫마디와 의미 대조
- [ ] 대조 결과 같은 이슈 판단 → 이전 takeaway 포함한 reflection 질문
- [ ] 미매칭 또는 빈 로그 → S1 바로 진행
- [ ] 같은 persona + 다른 테마 = 통과 (다른 이슈)
- [ ] 페르소나 2개+ → 선택 UI, 0개 → `/say-it-build` 안내, BLOCKED → 차단 안내
