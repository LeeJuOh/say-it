# 유저 안전 고지문 (SAFETY)

## What to build

마켓 배포되는 MVP 스킬에 **유저가 한눈에 보는 안전 고지문 한 장**(`SAFETY.md` 또는 동등 고지)을 포함. 우리는 ex-skill보다 **감정 위험이 큼** — 일부러 감정을 들쑤시고(S1 분출) 디스트레스/자해 신호를 다룸(ADR 0003) + 남이 받아 쓰는 배포물. 주의의무가 더 무거움.

**핵심: 새 결정 아님.** 실질 안전 요소는 이미 다 정해졌고 문서에 흩어져 있음. 이 슬라이스는 그걸 **유저용 한 장으로 모으고, 진입 시 surfacing**하는 것.

|---|---|
| "치료/상담 아님, 가벼운 한 판" | PRD 프레임 |

| 위기 핫라인 + 급성 위해 중단 | [ADR 0003](../adr/0003-safety-stop-hard-gate.md) + [이슈 07](07-distress-breaker.md) |
| 서술only · 데이터 로컬 | [ADR 0002](../adr/0002-narration-only-input.md) |
| 대상 = 산 사람 (고인·미성년 제외) | PRD 범위 |

### 어디서 보이나

- `/say-it-build` 첫 진입 시 **핵심 고지 1회** (치료 아님 + "내 인식 속 그 사람" + 힘들면 핫라인).
- 스킬 `references/`에 `SAFETY.md` 번들 ([이슈 01](01-hook-infra.md) scaffold가 태움).
- `/say-it` 세션 중에도 접근 가능 (핫라인은 이슈 07 디스트레스 경로가 런타임에 띄움 — 이 문서는 정적 고지).

### 네이밍 = SAFETY (not COMPLIANCE)

ex는 `COMPLIANCE.md`(데이터 준법 강조 — 그들 위험=프라이버시). 우리 위험 = **감정 안전**(서술only라 데이터 위험은 작음). 그래서 SAFETY. 구조는 ex COMPLIANCE.md 참고(데이터 처리 / 허용·금지 용도 / 심리건강 경고).

### 범위 밖 (앱 단계)

정식 약관(ToS)·PIPA 비식별·서버 저장 = 호스팅 앱 단계. 이 슬라이스는 **계약 문서 아니라 고지/면책/가이드 한 장**.

## Acceptance criteria

- [ ] `SAFETY.md` 작성 — 치료아님 / 한계 / 위기 핫라인 / 데이터(서술only·로컬) / 적정·금지 용도 / 대상범위 한 장에 모음
- [ ] `/say-it-build` 첫 진입 시 핵심 고지 1회 표시 (치료아님 + 내 인식 속 그 사람 + 힘들면 핫라인)
- [ ] 위기 핫라인 = **한국어 로케일** 번호 ([ADR 0003](../adr/0003-safety-stop-hard-gate.md) 언어의존), 이슈 07과 동일 번호 소스 공유 (불일치 금지)
- [ ] "치료/상담 아님" 프레임과 "핫라인 안내" 공존 명시 (핫라인 = 한계 인정·주의의무지 치료 아님 — ADR 0003)
- [ ] 대상 = 산 사람, 고인·미성년 제외 명시
- [ ] `references/`에 SAFETY.md 번들 (이슈 01 scaffold 목록에 추가)
- [ ] 정식 약관(ToS)·PIPA는 이 슬라이스 범위 밖 (앱 단계) 명시
